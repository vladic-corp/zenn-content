新潟でホームページ制作をやっている個人開発者です。先日、**AIエージェント向けに日本の法人データを従量課金で売るAPI**を作って公開しました。すると掲載から間もなく、海外のエージェントが誰の手も介さず自動で買っていきました。金額はごくわずかですが、「人間が一度も介在しない販売」が成立した瞬間でした。

この記事では、作るときに考えたことと、実装でハマったことをまとめます。

## x402とは何か（30秒版）

[x402](https://x402.org)は、HTTPの`402 Payment Required`を使った決済プロトコルです。エージェントがAPIを叩く→サーバーが402と価格を返す→エージェントのウォレットがUSDCで署名→再送→データが返る。**APIキーもアカウント登録も不要**で、「支払うウォレット＝アカウント」になります。Coinbaseが主導し、現在はLinux Foundation傘下。AWS・Google・Visa等40社が参加しています。

## 何を作るか

x402はまだ新しい決済で、対応サービスもそれほど多くありません。いくつか実際に触ってみて感じたのは、**LLMが原理的に答えを保証できない「公的な一次データの決定的な検証」**に価値が出そうだ、ということでした。日本の公的データは、エージェントから使える形にはほとんど整備されていません。そこで「日本の法人データ」を選びました。

## 作ったもの：Kaisha API

https://kaisha-api.hp-vladic.workers.dev

- **法人解決** `/resolve` — 社名（表記ゆれ吸収）or 法人番号13桁 → 正式名称・所在地・法人種別・存続状態。国税庁の法人番号公表データ**全576万社**
- **インボイス照合** `/invoice/verify` — 適格請求書の登録番号（T+13桁）を国税庁の公表データ**505万件**と実在照合。これは**x402上で世界唯一**のはず（LLMには原理的にできない決定的検証です）
- **営業日・祝日** `/calendar/*` — 内閣府データで営業日計算
- 価格は1コール$0.002〜0.01。USDCで**BaseとSolana両対応**

スタックはCloudflare Workers + Hono + D1。決済は`@x402/hono`にミドルウェアを1個挟むだけです。

## ハマりどころ（ここが本題）

### D1のSQL文上限は「10万バイト」

576万行のINSERTを流し込むとき、500行ずつに分割したのに`SQLITE_TOOBIG`。原因は**文字数ではなくUTF-8バイト数**の上限でした。日本語は1文字3バイトなので、文字数カウントは罠です。80KBバイトで切って解決。

### D1無料枠は「1DBあたり500MB」

全国データは約1GB。月$5のWorkers Paidにすれば済む話ですが、意地で**6DB分割**（法人×4・インボイス×2）にして固定費ゼロを維持しました。シャードキーは「番号の末尾2桁 % N」。末尾1桁だと10種類を4分割することになり30/30/20/20に偏るので、2桁（00-99）を使うのがポイントです。

### Solanaの売り手は「ATA」を作っておかないと全決済が落ちる

これが一番の学びでした。x402でSolana対応を宣言しても、**受け取りウォレットにUSDCのトークン口座（ATA）が無いと、全部の支払いがオンチェーンシミュレーションで失敗します**（`InstructionError: InvalidAccountData`）。エラーメッセージからは原因が読み取りにくいので、Solana対応する売り手は最初に自分宛てにUSDCを1円分送るか、ATAを明示的に作成しておきましょう。

### x402scanへの掲載は `/openapi.json` が必須

`.well-known/x402`は旧式扱いで、いまは**OpenAPI文書をオリジン直下に置く**のが登録要件です。無料エンドポイントには`"security": []`を付けないと「402を返さない」と怒られます。登録すると各エンドポイントを1本ずつ実際に402ハンドシェイクで検証されます（ちゃんとしてる）。

## 初売上の中身

掲載から間もなく、海外のウォレットが**安い順に全エンドポイントを1回ずつ**買っていきました。新着APIを自動で評価して回っているエージェント運用者のようです。エージェント経済の買い手は、こうやって自動的に新商品を見つけて試すのだと実感しました。

## 正直な現実と、それでもやる理由

- 市場はまだ立ち上がったばかりで、似たことを考える人も当然います
- それでも、決定的なデータ検証やMCPでの配布まで含めて作り込めば、差別化はできると思っています
- 固定費0円・限界費用ほぼ0円なので、これは「エージェント経済が来るかどうか」への**極めて安いコールオプション**です

## エージェントから使う方法

MCP対応クライアント（Claude Code等）なら1行です：

```bash
claude mcp add kaisha -e EVM_PRIVATE_KEY=0x<USDC入りBase財布> -- npx -y kaisha-mcp
```

パッケージはこちら：https://www.npmjs.com/package/kaisha-mcp
